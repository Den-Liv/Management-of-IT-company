## Описание добавленного функционала

#### 1. Подсистема «Обслуживание клиентов»

Подсистема автоматизирует процесс приема и выполнения заявок от клиентов, а также выставления счетов за оказанные услуги.

#### 1.1. Документ «Обслуживание клиентов»

*   **Назначение:** Учет обращений клиентов и фиксация выполненных специалистами работ.
*   **Процесс работы:**
    1.  Менеджер принимает звонок и создает заявку на выезд специалиста, фиксируя проблему клиента.
    2.  Менеджер планирует дату и время выполнения заявки. При планировании **Telegram-бот** автоматически оповещает специалистов о новой задаче.
    3.  В назначенное время специалист выполняет работы (на объекте клиента или удаленно).
    4.  По итогам работ клиент подписывает **лист учета рабочего времени**, где перечислены виды работ и часы к оплате.
    5.  Специалист вносит данные в систему:
        *   Перечень выполненных работ.
        *   Фактически потраченное время.
        *   Количество часов к оплате клиентом.
        *   Прикрепляет скан/фото подписанного листа учета.
*   **Пользователи и права доступа:**
    *   Документ вводят **менеджеры** и **специалисты**.
    *   Для работы созданы предопределенные профили: `Специалист`, `Менеджер`.

#### 1.2. Документ «Реализация товаров и услуг»

*   **Назначение:** Выставление счетов клиентам (актов об оказании услуг).
*   **Новый функционал:**
    *   Добавлена поддержка договоров с видом «Абонентская плата».
    *   Реализовано **автоматическое заполнение** документа суммой ежемесячной абонентской платы, а также суммой за выполненные работы (на основе данных из документов «Обслуживание клиентов» за месяц).
    *   Из документа формируется печатная форма «Акт об оказанных услугах».
*   **Пользователи и права доступа:**
    *   Документ использует **бухгалтер**.
    *   Для работы создан предопределенный профиль: `Бухгалтер`.

#### 1.3. Обработка «Массовое создание документов Реализация товаров и услуг»

*   **Назначение:** Автоматизация ежемесячной процедуры выставления актов.
*   **Описание:** В начале каждого месяца бухгалтер запускает обработку для формирования актов по всем абонентским договорам.
*   **Пользователи и права доступа:**
    *   Обработку использует **бухгалтер**.
    *   Для работы создана отдельная роль `Использование обработки массовое создание актов`, которая включена в профиль `Бухгалтер`.

#### 2. Подсистема «Расчёт управленческой зарплаты»

Подсистема предназначена для кадрового учета и расчета заработной платы по специфической схеме предприятия.

*   **Категории сотрудников и схема оплаты:**
    *   **Специалисты:** Минимальный оклад + процент от стоимости выполненных ими работ.
    *   **Административный персонал:** Фиксированный оклад.
    *   **Общее для всех:** Возможность начисления разовой фиксированной премии по решению руководства. Обязательное удержание НДФЛ (13%).

#### 2.1. Документы для расчета зарплаты

1.  **«Начисление заработной платы»** — для расчета оклада и процентов специалистам.
2.  **«Начисление фиксированной премии»** — для начисления разовых поощрений.

#### 2.2. Планирование отпусков

*   **Документ «Планирование отпусков»:** Позволяет составлять ежегодный график отпусков. Данные вносятся в начале года.
*   **Контроль пересечений:** Для визуального анализа загруженности сотрудников реализована кнопка **«Анализ графика»**. При нажатии открывается форма с **диаграммой Ганта**, отображающей периоды отпусков всех сотрудников для выявления пересечений.

#### 2.3. Пользователи и права доступа

*   Для работы с подсистемой создан предопределенный профиль пользователей: `Кадровик-расчетчик`.

### 3. Отчеты

В систему добавлены новые отчеты для анализа деятельности компании.

| Название отчета | Описание | Ценность для бизнеса |
| :--- | :--- | :--- |
| **Выработка специалистов** | Показывает, сколько часов за период отработал конкретный специалист и какая сумма процентов ему за этот период начислена. | Анализ эффективности и расчет переменной части зарплаты. |
| **Анализ выставленных актов** | Содержит информацию о клиентах и договорах, а также о суммах, которые должны быть выставлены к оплате на основе закрытых заявок (документы «Обслуживание»). | Контроль дебиторской задолженности и своевременности выставления счетов. |
| **Расчеты с сотрудниками** | Представляет детальную информацию о начисленных и выплаченных суммах каждому сотруднику за период, а также о задолженности (на начало и конец периода). | Прозрачность взаиморасчетов с персоналом. |
| **Расход запланированных отпусков** | Пользователь выбирает год. Отчет показывает, сколько дней отпуска у сотрудника было запланировано и сколько дней он фактически отгулял в выбранный период. | Контроль исполнения графика отпусков и накопления неиспользованных дней. |
